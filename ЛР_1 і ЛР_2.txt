1.Git — розподілена система керування версіями файлів та спільної роботи.

2.В першу чергу всі файли поділяються на відстежувані (tracked) і не відстежувані (untracked). Untracked - це новий файл, який не був доданий для відстеження командою git add. Всі інші файли є tracked.

3.Команда git add додає вміст робочої директорії в індекс (staging area) для подальшого комміта.

4.Команда git status показує стан файлів в робочій директорії і індексі: які файли змінені, але не доданы в індекс; які очікують комміта в індексі.

5.Команда git commit бере всі дані, додані в індекс за допомогою git add, і зберігає їх зліпок у внутрішній базі даних, а потім зрушує покажчик поточної гілки на цей зліпок.

6.Щоб взяти директорію та перетворити її на новий репозиторій Git, щоб ви могли почати керувати її версіями, ви можете просто виконати git init.

7.Команда git checkout використовується для переключення гілок та отримання вмісту до вашої робочої директорії.

8.Команда git reset переважно використовується для скасування речей, як ви напевно можете здогадатись через дієслово reset.

9.Команда git log використовується, щоб показати досяжну записану історію проекту, починаючи з останнього відбитку коміту у зворотному порядку.

10.Команда git status показує стан файлів в робочій директорії і індексі: які файли змінені, але не доданы в індекс; які очікують комміта в індексі.

13.Команда git branch насправді є чимось на кшталт інструменту керування гілками. Вона може виводити список існуючих гілок, створювати нову гілку, вилучати гілки та перейменовувати їх.

16.Команда git pull є загалом комбінацією git fetch та git merge, тобто Git отримає зміни зі заданого віддаленого сховища, а потім одразу спробує злити їх до поточної гілки.
